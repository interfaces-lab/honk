Multikit Migration

Multikit is the shadcn-first component kit for Multi. It starts by copying the existing UI primitive source and consuming it internally before registry copy-out tooling is added.

Package

  • Source package: packages/multikit
  • Import path: @multi/multikit/*
  • Styles export: @multi/multikit/styles.css

CSS Rule

Global Multikit CSS is imported once at the app entry boundary. Page-level CSS imports are avoided unless a page owns an isolated third-party stylesheet or route-specific demo surface.

Distribution Direction

  • No provider-level theming API.
  • No theme builder.
  • Light and dark themes are CSS variable definitions.
  • Registry metadata starts in packages/multikit/registry.json.
