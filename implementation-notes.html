Implementation Notes

Sidebar Hover Actions

  • Cursor keeps sidebar row actions mounted and gates them with max-width, opacity, overflow, and pointer-events. The Multi row now mirrors that pattern instead of using visible always-on buttons.
  • The thread hover cluster is ordered as archive, time, pin. It remains hidden when the row is not hovered. It also appears on keyboard focus so the actions remain reachable without a mouse.
  • Pin state is local UI state, persisted with the other sidebar UI state. Pinned threads now render in a separate Pinned group and are removed from their original project group while pinned. The pinned group keeps timestamp ordering, newest first.
  • The pinned group is treated as a synthetic sidebar section. It keeps bulk thread actions such as mark-read and archive, but does not show project-only affordances like creating a new agent in that group or opening the group in an editor window.

Chrome Spacing

  • Cursor's no-titlebar chrome uses a 34px top strip. Multi's workbench top strip now uses 34px, and the titlebar control row also uses the same 34px calculation so the left and right sidebar toggles share Cursor's centered 22px control rhythm.
  • The Electron traffic-light sidebar padding default is reduced from 28px to 26px. With the sidebar header's own 8px top padding, the first sidebar control starts at the same 34px top-chrome boundary Cursor uses for no-titlebar chrome.

Icon Choices

  • Archive actions use IconArchive1.
  • Pin and unpin use IconPin and IconUnpin.
  • Git discard and settings reset affordances use IconStepBack.
