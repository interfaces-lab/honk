# Code-quality audit — `big-refactor` branch

Scope: 166 changed source files vs `main`, grouped into 25 subsystem clusters. Each cluster got a deep structural reviewer; every blocker/major finding was then handed to an adversarial verifier instructed to refute it. **63 agents, 3.2M tokens.**

**Verified outcome:** 1 blocker, 17 majors, 56 minors survived. Of the 37 high-severity claims sent to verification, 21 were CONFIRMED, 15 were confirmed-but-tempered, and 1 was refuted and dropped.

---

## Systemic patterns (the real story)

These recur across many files — each is one design-level problem, not N scattered nits.

### 1. Bespoke helpers reimplement a utility the repo already owns.

Small functions re-derive something canonical instead of importing the existing owner, so the same logic lives in two-plus places and can silently drift. Basename/path parsing, Option unwrapping, port reservation, secret hashing, base64/UTF-8 encoding, FNV hashing, and the foreground predicate all got hand-rolled copies during the refactor.

**Files (13):** `packages/desktop/src/app/desktop-app.ts`, `packages/app/src/tool-message.tsx`, `packages/app/src/onboarding.tsx`, `packages/app/src/directory-picker.tsx`, `packages/app/src/appearance-store.ts`, `packages/app/src/notification-permission.ts`, `packages/app/src/toast-store.ts`, `packages/desktop/src/backend/opencode-sidecar.ts`, `packages/cli/src/host.ts`, `packages/cli/src/auth.ts`, `packages/opencode/src/connection.ts`, `packages/mobile/src/remote.tsx`, `packages/mobile/src/thread-screen.tsx`

**Fix:** Make 'grep for the canonical owner before writing a utility' a hard rule (it is already in project memory). Delete each copy and import from its owner: basename/isWindowsDrivePath from @honk/shared/paths, Option.getOrUndefined from effect, NetService from @honk/shared/Net, TextEncoder/btoa for base64, one hash+constant-time-compare pair in auth.ts, one isWindowForeground, one cssVarName exported from @honk/ui.

### 2. Untrusted-input decoding is hand-rolled with JSON.

parse + Reflect.get + typeof guards instead of the repo's Effect Schema decoders, even where a matching Schema.Struct already exists. Each boundary re-validates field-by-field with bespoke error strings rather than declaring the shape once and decoding through the canonical string decoders.

**Files (5):** `packages/desktop/src/aux/git.ts`, `packages/desktop/src/aux/server.ts`, `packages/mobile/src/remote.tsx`, `packages/cli/src/bin.ts`, `packages/shared/src/desktop-api.ts`

**Fix:** Route every boundary through Schema: use fromLenientJson/decodeJsonResult for JSON strings, import the seven Git*Input schemas from @honk/shared/git for the aux routes, define StoredRegistry/Credential/Draft structs for mobile, and derive DTO types from their schema (type X = typeof XSchema.Type) so there is one source of truth instead of a parallel interface.

### 3. Multi-concern giant files.

The refactor fused unrelated responsibilities into single modules well past the ~1000-line smell line, mixing pure parsers/state machines/embedded scripts with the service orchestration around them. Most of the buried blocks are already free functions that move verbatim.

**Files (8):** `packages/desktop/src/aux/git.ts`, `packages/desktop/src/browser/browser-automation.ts`, `packages/opencode/src/client.ts`, `packages/ui/src/tabs.tsx`, `packages/app/src/watch-registry.ts`, `packages/desktop/src/window/desktop-window.ts`, `packages/desktop/src/aux/settings.ts`, `packages/app/src/thread/transcript.tsx`

**Fix:** Cleave each file at its natural seam with identical signatures: pure parsers and schemas into sibling modules, drag/reconnect/geometry controllers into hooks or services, embedded browser JS into a *-injected-scripts module. Treat >1000 lines as a blocker unless the block genuinely closes over the surrounding closure.

### 4. Scattered conditional dispatch instead of one typed table or discriminated union.

The same enumerable decision (a status, a section, a tool category, a filter, a controller-bound-or-not fork) is expanded inline as a nested ternary or a family of parallel switch functions, so every case appears in multiple places and must be kept consistent by hand.

**Files (9):** `packages/app/src/tab-store.ts`, `packages/app/src/home.tsx`, `packages/app/src/update-pill.tsx`, `packages/app/src/settings.tsx`, `packages/app/src/composer/home-composer.tsx`, `packages/app/src/toast-store.ts`, `packages/mobile/src/conversation.tsx`, `packages/mobile/app/(tabs)/home/index.tsx`, `packages/desktop/src/browser/browser-automation.ts`

**Fix:** Push each into a single structure: a Record<Key, handler> dispatch table (settings sections, mobile status filter, tool verb rows), one pure selector computed in a single pass (update-pill view, home-composer worktree status), a discriminated union for mutually-exclusive state (home.tsx selection, toast timer entry), and one resolved mutation target for tab-store's controller===null forks.

### 5. Copy-pasted twin functions and pipelines that differ by a single datum.

Two entry points, two loaders, two emitters, or two config pipelines share a body verbatim and diverge only in a label, key name, or path, so a fix has to be made in both.

**Files (9):** `packages/app/src/thread-notifications.ts`, `packages/opencode/src/connection.ts`, `packages/opencode/src/client.ts`, `packages/desktop/src/aux/settings.ts`, `packages/desktop/src/backend/opencode-sidecar.ts`, `packages/app/src/connection-store.ts`, `packages/desktop/src/browser/browser-automation.ts`, `packages/ui/src/alert-dialog.tsx`, `packages/app/src/workbench-terminal.tsx`

**Fix:** Collapse each pair to one parameterized function plus data: emitThreadSignal, namedSecretFromUrl(key), a generic paginate<T> and WatchedConfigFile<T>, one sidecarFetch, one connectAndBind tail, one browser element-resolver, one modal-surface builder, one startSession closure.

### 6. Duplicated presentation recipes and build pins in @honk/ui with no shared source.

The overlay/modal surface, the themed shell frame, the input-surface geometry, per-slot passthrough wrappers, and the lightningcss browser-targets pin are each re-declared in every consumer instead of living in one style module.

**Files (9):** `packages/ui/src/dialog.tsx`, `packages/ui/src/alert-dialog.tsx`, `packages/ui/src/menu.tsx`, `packages/ui/src/text-field/index.web.tsx`, `packages/ui/src/field.tsx`, `packages/app/src/shell.tsx`, `packages/app/src/boot.tsx`, `packages/app/vite.config.ts`, `packages/ui/vite.config.ts`

**Fix:** Add shared style/factory modules that own the invariants: overlaySurface+overlayMotion+RING constants, a styledSlot factory for leaf wrappers, one input-surface primitive shared by TextField and Field, a ThemedShell wrapper, and a honkStylexPlugin()/stylexLightningcssTargets export imported by both vite configs.

### 7. Dead and premature scaffolding carried forward through the refactor.

Whole subsystems and shapes survive as plumbing with no live consumer: a two-process coordinator managing one child, an unreachable submenu stack, the old protobuf oneof tool shape, orphaned onboarding styles, an unused row component, a phantom sequence guard, and validated documentation-as-code.

**Files (9):** `packages/desktop/scripts/dev.ts`, `packages/app/src/command-menu-store.ts`, `packages/app/src/command-menu-model.ts`, `packages/app/src/command-menu.tsx`, `packages/marketing/src/components/product-frame/demo-animation.ts`, `packages/app/src/onboarding.tsx`, `packages/mobile/src/project-row.tsx`, `packages/app/src/workbench-changes.tsx`, `packages/ui/src/theme.ts`

**Fix:** Delete it now and reintroduce a single typed frame when a real consumer lands: collapse dev.ts to a single-child model, remove the submenu concept across all three command-menu files, flatten the marketing tool item to flat fields, drop the orphaned onboarding style keys, delete project-row.tsx and the workbench-changes sequence guard, and remove colorProvenance (theme-parity.json already guards completeness).

### 8. Reconnect/event-pump state machines and abortable-delay loops reimplemented per consumer.

The same 'consume events with backoff, reconnect, cancel on abort' concern is written inline in several places with its own generation counter and constants rather than being provided once by the client layer.

**Files (4):** `packages/app/src/watch-registry.ts`, `packages/mobile/src/thread-screen.tsx`, `packages/mobile/src/remote.tsx`, `packages/opencode/src/client.ts`

**Fix:** Hoist a shared delay(signal, ms) and move the resilient event stream into @honk/opencode as an async-iterable that yields through reconnects, so watch-registry and the mobile screens subscribe to one backoff-owning stream instead of each hand-rolling a while-loop.

## Top code-judo moves (highest leverage)

### 1. Shatter git.ts and route GitHub JSON through canonical schemas

**Files:** `packages/desktop/src/aux/git.ts`, `packages/shared/src/git.ts`

Split the 3853-line god-module into run-process.ts, git-porcelain.ts (the pure parsers, which move verbatim), github-pr.ts, and git-status-watch.ts, leaving DesktopAuxGitService as a thin orchestrator. In the same pass, delete the hand-wired JSON.parse + Schema.decodeUnknownOption + Option plumbing for the gh CLI and decode raw stdout through fromLenientJson, collapsing parseGitHubRepositoryCloneUrls' double-decode to one. Deletes the 'one giant aux file' shape and the parallel decode path.

_Impact: Removes ~3000 lines from a single file, deletes decodeGitHubJsonRecord and the ad-hoc Option branches, and makes the pure parsers/schemas independently testable and reusable in @honk/shared/git._

### 2. One overlay/modal-surface style module collapses the whole floating-primitive family

**Files:** `packages/ui/src/dialog.tsx`, `packages/ui/src/alert-dialog.tsx`, `packages/ui/src/menu.tsx`, `packages/ui/src/text-field/index.web.tsx`, `packages/ui/src/field.tsx`

Extract overlaySurface + overlayMotion + the RING constants into one style module every popup composes, factor the shared modal wrapper set (Popup/Title/Description/Header/Footer) into one internal builder instantiated for Dialog and AlertDialog, replace the ~18 hand-rolled themed-slot wrappers with a styledSlot factory, and share the input-surface geometry between TextField and Field. Each primitive keeps only its genuine deltas.

_Impact: Deletes six-plus verbatim overlay re-declarations, the near-total alert-dialog clone, ~18 passthrough wrappers, and one duplicated input surface, so the design invariant lives in one place instead of drifting per primitive._

### 3. Introduce createPersistedStore and retire the hand-rolled external stores

**Files:** `packages/app/src/modes.ts`, `packages/app/src/presets.ts`, `packages/app/src/toast-store.ts`, `packages/app/src/tab-store.ts`

Extract one createPersistedStore<T>({ key, parse, fallback }) that owns localStorage hydrate/persist/SSR/try-catch, the listener fan-out, and the useSyncExternalStore hook; modes.ts and presets.ts shrink to their value type, guard, and default. Apply repo-wide (it recurs across ~7 persisted and ~21 total external stores) so it becomes the house pattern rather than a cluster-local one-off.

_Impact: Removes the duplicated store boilerplate from many modules and gives the app one audited persistence/subscription implementation instead of 20-plus bespoke ones._

### 4. Delete the carried-forward dead scaffolding in one sweep

**Files:** `packages/desktop/scripts/dev.ts`, `packages/app/src/command-menu-store.ts`, `packages/app/src/command-menu.tsx`, `packages/marketing/src/components/product-frame/demo-animation.ts`, `packages/app/src/onboarding.tsx`, `packages/mobile/src/project-row.tsx`, `packages/ui/src/theme.ts`, `packages/app/src/workbench-changes.tsx`

Remove every subsystem and shape that survives without a live consumer: collapse dev.ts to a single-child model (drop bothChildrenSettled/beginPeerShutdown/the peer vocabulary), delete the entire submenu push/pop concept threaded through all three command-menu files, flatten the marketing protobuf oneof tool shape to flat fields and drop the dead grep/output payloads, delete the orphaned onboarding demo/intro/provider styles and keyframes, delete unused project-row.tsx, remove colorProvenance, and drop workbench-changes' phantom sequence guard.

_Impact: Deletes roughly a thousand lines of plumbing across five packages and removes whole concepts (peer-shutdown, submenus, the oneof tool model, documentation-as-code) that readers would otherwise have to reason about._

### 5. Reunite validators and grammars with the shared layer that owns their types

**Files:** `packages/desktop/src/aux/server.ts`, `packages/desktop/src/aux/settings.ts`, `packages/desktop/src/aux/git.ts`, `packages/mobile/src/remote.tsx`, `packages/cli/src/host.ts`, `packages/cli/src/auth.ts`, `packages/cli/src/bin.ts`

Push feature logic out of the leaf files back to the module that owns each concept: replace the seven hand-rolled git-input validators with the existing decodeGitXxx one-liners (deleting optionalNumber/optionalStringArray), move the entire keybinding grammar (parser/encoder/when-expression lexer + rule compiler + default-merge) from settings.ts into @honk/shared/keybindings alongside its AST types, decode the mobile persisted registry through hoisted shared Schemas, and unify the CLI's secret hash + URL-shape contract into single canonical functions (auth.ts / state.ts).

_Impact: Deletes ~120 lines of duplicated git validators plus optionalNumber/optionalStringArray, drops settings.ts well under the size threshold by relocating its pure grammar, and removes the drift-prone second definitions of secret hashing and the public-URL invariant._

### 6. Replace the scattered conditional dispatch with typed tables and unions

**Files:** `packages/app/src/tab-store.ts`, `packages/app/src/home.tsx`, `packages/app/src/update-pill.tsx`, `packages/app/src/settings.tsx`, `packages/mobile/src/conversation.tsx`, `packages/app/src/composer/home-composer.tsx`

Turn each enumerable decision into one structure: resolve tab-store's mutation target once (a mutations object that is controller.actions or a single unbound adapter) to delete all thirteen controller===null forks and the duplicated reorder math; model home.tsx's mutually-exclusive selection as one discriminated union instead of two reconciled useState fields; collapse update-pill's four switch functions into one deriveUpdatePillView selector; make settings sections a Record<SectionId, Component>; and back the conversation tool-verb and home-composer worktree-status ternaries with small lookup tables.

_Impact: Removes thirteen duplicated store forks, five-plus nested/parallel ternaries and the cross-reset dance in home.tsx, so each status/section/category is declared exactly once and exhaustiveness is checked by the type._
