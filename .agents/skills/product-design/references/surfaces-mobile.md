# Mobile

Mobile shares the product objects, capabilities, status vocabulary, copy dial, and token contract with
desktop; it does not inherit desktop window chrome, keyboard shortcuts, hover behavior, or the 8px inset
recipe automatically. Use native navigation, focus, safe-area, keyboard, and touch conventions from the
app's owning mobile guidance. Verify touch targets, software-keyboard behavior, compact width, and long
content on the actual mobile surface.

Current accepted evidence does not settle mobile-specific conversation layout, navigation hierarchy,
or queue presentation. Keep these in `coverage-gaps.md` until verified rather than porting desktop UI.
